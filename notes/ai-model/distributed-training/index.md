# 分布式训练专题

分布式训练是现代深度学习的核心技术之一。随着大语言模型（LLM）参数规模从亿级增长到千亿级甚至万亿级，单卡训练已经无法满足需求，分布式训练成为必然选择。

## 为什么需要分布式训练？

### 模型规模的挑战

近年来，模型参数量呈指数级增长：

| 模型 | 参数量 | 发布年份 |
|------|--------|----------|
| BERT-Large | 340M | 2018 |
| GPT-2 | 1.5B | 2019 |
| GPT-3 | 175B | 2020 |
| PaLM | 540B | 2022 |
| GPT-4 | ~1.8T (推测) | 2023 |

### 显存瓶颈

单张GPU显存有限，难以容纳大模型：

- **A100 (80GB)**：最多约 24B 参数（FP16）
- **H100 (80GB)**：最多约 24B 参数（FP16）
- 175B 模型需要约 350GB 显存（仅存储参数）

### 训练时间瓶颈

单卡训练大模型时间过长：

$$T_{\text{train}} = \frac{N_{\text{tokens}} \times N_{\text{params}} \times C}{\text{Throughput}}$$

其中：
- $N_{\text{tokens}}$：训练 token 数量
- $N_{\text{params}}$：参数数量
- $C$：每个参数处理每个 token 的计算量（约 6 FLOPs）

::: tip 估算示例
GPT-3 训练：
- 175B 参数，300B tokens
- 总计算量：$175 \times 10^9 \times 300 \times 10^9 \times 6 \approx 3.15 \times 10^{23}$ FLOPs
- A100 峰值算力 312 TFLOPS (FP16/BF16 with Tensor Core)
- 单卡训练时间：$\frac{3.15 \times 10^{23}}{312 \times 10^{12} \times 3600} \approx 280,000$ 小时 ≈ 32 年！
:::

## 分布式训练核心策略

分布式训练主要有以下几种并行策略：

```
┌─────────────────────────────────────────────────────────────────┐
│                      分布式训练策略全景                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌─────────────────┐  ┌─────────────────┐  ┌─────────────────┐ │
│  │   数据并行 DP    │  │   模型并行 MP    │  │   流水线并行 PP  │ │
│  │  (Data Parallel)│  │ (Model Parallel)│  │(Pipeline Parallel)│
│  └────────┬────────┘  └────────┬────────┘  └────────┬────────┘ │
│           │                    │                    │          │
│           ▼                    ▼                    ▼          │
│  ┌─────────────────┐  ┌─────────────────┐  ┌─────────────────┐ │
│  │ 每卡完整模型副本 │  │ 模型切分到多卡  │  │ 层间切分到多卡  │ │
│  │ 数据分片处理    │  │ 层内切分       │  │ 流水执行       │ │
│  └─────────────────┘  └─────────────────┘  └─────────────────┘ │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────────┐│
│  │                     组合并行策略                            ││
│  │   3D并行 = DP + TP + PP                                     ││
│  │   ZeRO = DP的显存优化增强                                   ││
│  └─────────────────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────────────────┘
```

### 1. 数据并行 (Data Parallelism, DP)

每个设备持有**完整的模型副本**，数据集被分割到不同设备上。

- ✅ 实现简单，易于使用
- ✅ 通信开销相对较小
- ❌ 每卡需要存储完整模型，显存占用大
- ❌ 模型不能超过单卡显存

**代表**：PyTorch DDP (DistributedDataParallel)

### 2. 张量并行 (Tensor Parallelism, TP)

将模型**单层的参数切分**到多个设备上，层内并行计算。

- ✅ 突破单卡显存限制
- ✅ 通信与计算可重叠
- ❌ 需要高频通信（All-Reduce）
- ❌ 对通信带宽要求高

**代表**：Megatron-LM

### 3. 流水线并行 (Pipeline Parallelism, PP)

将模型的**不同层切分**到不同设备上，形成流水线执行。

- ✅ 减少显存占用
- ✅ 通信量较小
- ❌ 存在"气泡"（空闲时间）
- ❌ 需要精细调度

**代表**：GPipe, PipeDream

### 4. ZeRO 优化

ZeRO (Zero Redundancy Optimizer) 是数据并行的显存优化技术，通过**消除冗余存储**大幅降低显存占用。

- ✅ 显存占用大幅降低
- ✅ 可训练超大模型
- ❌ 通信量增加
- ❌ 实现复杂

**代表**：DeepSpeed ZeRO

## 三维并行 (3D Parallelism)

现代大模型训练通常组合多种并行策略：

```
┌─────────────────────────────────────────────────────────────────┐
│                     3D 并行示意图                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│     Pipeline Stage 0      Pipeline Stage 1      Pipeline Stage 2│
│    ┌─────────────────┐   ┌─────────────────┐   ┌─────────────┐ │
│    │ TP0  TP1  TP2   │   │ TP0  TP1  TP2   │   │ TP0  TP1   │ │
│    │  │    │    │    │   │  │    │    │    │   │  │    │    │ │
│    │ Layer 0-5      │──▶│ Layer 6-11     │──▶│ Layer 12-17│ │
│    └─────────────────┘   └─────────────────┘   └─────────────┘ │
│            │                    │                    │          │
│            ▼                    ▼                    ▼          │
│    ┌─────────────────────────────────────────────────────────────┐│
│    │              数据并行维度 (DP)                              ││
│    │   多组相同的 3D 并行组同时处理不同数据                      ││
│    └─────────────────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────────────────┘
```

**并行配置示例**（以 64 卡训练为例）：

| 配置 | DP | TP | PP | 适用场景 |
|------|----|----|----|---------|
| 中等模型 | 8 | 4 | 2 | 7B-13B |
| 大模型 | 4 | 8 | 2 | 70B-175B |
| 超大模型 | 1 | 8 | 8 | 175B+ |

## 显存占用对比

不同并行策略的显存占用（以 $N$ 卡训练参数量 $\Psi$ 的模型为例）：

| 策略 | 模型状态显存 | 优化器状态显存 | 梯度显存 | 总计 |
|------|-------------|---------------|---------|------|
| 单卡 | $\Psi$ | $2\Psi$ (Adam) | $\Psi$ | $4\Psi$ |
| DP | $\Psi$ | $2\Psi$ | $\Psi$ | $4\Psi$ (每卡) |
| TP (N卡) | $\Psi/N$ | $2\Psi/N$ | $\Psi/N$ | $4\Psi/N$ |
| PP (N卡) | $\Psi/N$ | $2\Psi/N$ | $\Psi/N$ | $4\Psi/N$ |
| ZeRO-3 (N卡) | $\Psi/N$ | $2\Psi/N$ | $\Psi/N$ | $4\Psi/N$ |

::: warning 注意
实际显存还需要考虑：
- 激活值（Activation）
- 临时缓存
- 通信缓冲区
:::
