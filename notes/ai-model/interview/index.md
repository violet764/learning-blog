## 🗺️ 学习路径建议

### 路径一：校招/实习准备

适合基础较好，希望系统复习的应届生。

```
Week 1: 深度学习基础
├── 神经网络基础（前向传播、反向传播）
├── 优化器（SGD、Adam、AdamW）
├── 激活函数（ReLU、GELU、SwiGLU）
└── 正则化（Dropout、LayerNorm）

Week 2: Transformer 核心
├── Attention 机制原理与计算
├── 位置编码（正弦、RoPE、ALiBi）
├── 多头注意力设计思想
└── Encoder vs Decoder

Week 3: 大模型原理
├── 预训练目标与数据处理
├── 微调方法（SFT、RLHF、DPO）
├── LoRA 原理与实现
└── 推理优化技术

Week 4: 实战准备
├── 项目经验梳理
├── 场景题目练习
└── 模拟面试
```

### 路径二：社招/跳槽准备

适合有一定工作经验，需要查漏补缺的工程师。

```
重点复习:
├── 深入理解原理（为什么这样设计）
├── 性能优化经验（实际做过什么优化）
├── 问题排查能力（遇到过什么问题，如何解决）
└── 系统设计能力（如何设计一个大模型应用）
```

## 💡 答题技巧

### 回答结构化

采用 **STAR** 或 **What-Why-How** 结构：

```
1. What（是什么）: 简明扼要定义概念
2. Why（为什么）: 解释设计动机和原因
3. How（怎么做）: 说明具体实现或应用
4. 追问（深入）: 准备好可能的追问点
```

### 示例：回答 "什么是 Attention 机制？"

```
【基础回答】
Attention 机制是一种让模型关注输入序列中重要部分的技术。
核心公式是 Attention(Q,K,V) = softmax(QK^T/√d)V

【深入回答 - 为什么需要】
传统的 RNN 处理长序列时存在信息丢失问题，Attention 允许
模型直接关注任意位置的信息，解决了长距离依赖问题。

【深入回答 - 设计细节】
除以 √d 是为了防止点积结果过大导致 softmax 梯度消失；
多头注意力让模型同时关注不同的表示子空间...

【实战经验】
在实际项目中，我使用过 FlashAttention 来加速训练，
可以将注意力计算的显存占用从 O(n²) 降低...
```

### 避免的问题

| 问题 | 后果 | 正确做法 |
|------|------|----------|
| 背诵答案 | 追问时卡住 | 理解原理，用自己的话表达 |
| 泛泛而谈 | 显得没有深度 | 结合具体例子和数据 |
| 不知道硬编 | 面试官一眼识破 | 坦诚承认，展示学习态度 |
| 偏题跑题 | 浪费时间 | 先回答核心，再展开细节 |

## ⚠️ 注意事项

::: warning 面试准备建议
1. **理解胜于背诵**：面试官会追问细节，背诵容易露馅
2. **实践结合理论**：有项目经验的回答更有说服力
3. **诚实面对盲区**：不知道的知识点坦诚说明，不要硬编
4. **展示思考过程**：即使答案不完全正确，清晰的思路也能加分
:::
