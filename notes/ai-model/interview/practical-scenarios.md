## 二、问题排查类

### Q4: 训练时出现 NaN 怎么办？

**参考回答：**

```
排查步骤:

1. 定位 NaN 出现的位置
   ├── 检查输入数据是否有 NaN/Inf
   ├── 检查各层输出是否有 NaN
   └── 检查梯度是否有 NaN

2. 常见原因分析
   ├── 学习率过大
   ├── 输入数据异常（超大值、NaN）
   ├── 除零错误
   ├── 损失函数计算问题
   └── 数值溢出

3. 解决方法
   ├── 降低学习率
   ├── 添加梯度裁剪
   ├── 检查数据预处理
   ├── 添加数值稳定性处理
   └── 使用混合精度训练的 loss scaling

4. 预防措施
   ├── 添加 NaN 检测 hook
   ├── 监控关键指标
   └── 数据质量检查
```

**代码示例**：

```python
# 检测 NaN
def check_nan(tensor, name):
    if torch.isnan(tensor).any() or torch.isinf(tensor).any():
        print(f"NaN/Inf detected in {name}")
        return True
    return False

# 训练循环中添加检查
loss = model(input)
if check_nan(loss, "loss"):
    break
```

### Q5: 模型生成内容质量不好怎么排查？

**参考回答：**

```
系统化排查:

1. 数据层面
   ├── 训练数据质量如何
   │   ├── 标注是否准确
   │   ├── 分布是否均衡
   │   └── 覆盖是否全面
   ├── 是否有数据泄露
   └── 测试数据是否有问题

2. 模型层面
   ├── 模型是否收敛
   ├── 是否过拟合/欠拟合
   ├── 是否有灾难性遗忘
   └── 对比基线模型

3. 解码层面
   ├── 解码策略是否合适
   ├── 温度/top-p 参数
   └── 是否有重复生成

4. 输入层面
   ├── Prompt 设计是否合理
   ├── 示例选择是否合适
   └── 输入格式是否正确

5. Case 分析
   ├── 收集典型 bad case
   ├── 分析失败模式
   └── 针对性改进
```

**追问：如何分析 bad case？**

```
分析步骤:
1. 分类失败类型
   - 事实错误
   - 逻辑错误
   - 格式错误
   - 不完整
   - 偏题

2. 寻找规律
   - 是否集中在某类问题
   - 是否与输入长度相关
   - 是否与特定 token 相关

3. 提出假设并验证
   - 假设: 长文本处理能力不足
   - 验证: 检查长文本任务表现
   - 结论: 确认/否定假设

4. 制定改进方案
   - 数据增强
   - 模型改进
   - Prompt 优化
```

### Q6: 推理速度太慢怎么优化？

**参考回答：**

```
优化分析框架:

1. 性能分析
   ├── 瓶颈在哪里
   │   ├── 计算密集（GPU 利用率高）
   │   ├── 内存密集（内存带宽瓶颈）
   │   └── IO 密集（数据传输瓶颈）
   └── 测量各阶段耗时

2. 优化策略
   ├── 计算优化
   │   ├── FlashAttention
   │   ├── 算子融合
   │   └── 推测解码
   │
   ├── 模型优化
   │   ├── 量化（INT8/INT4）
   │   ├── 剪枝
   │   └── 蒸馏
   │
   ├── 系统优化
   │   ├── KV Cache 优化
   │   ├── Continuous Batching
   │   └── 并行推理
   │
   └── 业务优化
       ├── 缓存策略
       ├── 异步处理
       └── 批量处理

3. 效果评估
   ├── 优化前后对比
   ├── 质量影响评估
   └── 成本效益分析
```

**追问：如何判断是计算密集还是内存密集？**

```
观察 GPU 指标:
- 计算密集: GPU 利用率 > 80%
- 内存密集: GPU 利用率低, 内存带宽饱和
- IO 密集: GPU 利用率波动大

针对优化:
- 计算密集: 算子优化、并行计算
- 内存密集: 量化、KV Cache 优化
- IO 密集: 数据预取、流水线
```

**追问：如何实现语义缓存？**

```python
class SemanticCache:
    def __init__(self, embedding_model, threshold=0.95):
        self.cache = {}  # embedding -> response
        self.embedding_model = embedding_model
        self.threshold = threshold
    
    def get(self, query):
        query_emb = self.embedding_model.encode(query)
        
        # 查找最相似的缓存
        for cached_emb, response in self.cache.items():
            similarity = cosine_similarity(query_emb, cached_emb)
            if similarity > self.threshold:
                return response  # 缓存命中
        
        return None  # 缓存未命中
    
    def set(self, query, response):
        query_emb = self.embedding_model.encode(query)
        self.cache[query_emb] = response
```

   - 如何帮助团队
   - 知识分享经验

## 📝 总结

### 实战问题回答技巧

| 类型 | 关键点 |
|------|--------|
| **项目经验** | STAR 框架、量化结果、个人贡献 |
| **问题排查** | 系统化思路、分析过程、预防措施 |
| **系统设计** | 架构图、关键决策、权衡分析 |
| **开放问题** | 结构化思考、个人观点、技术洞察 |
| **行为问题** | STAR 方法、反思总结、团队价值 |

### 常见追问应对

1. **"为什么这样选择？"**：说明决策依据，对比其他方案
2. **"如果重新设计？"**：展示反思能力，提出改进点
3. **"遇到过什么困难？"**：真实案例，展示解决问题能力
4. **"如何权衡？"**：明确约束条件，说明优先级判断
