### 1.2 GPT 架构核心设计

GPT 系列采用 **Decoder-only Transformer** 架构，核心组件包括：

```
输入 Token IDs
     ↓
┌─────────────────────────────────────────────────────────────┐
│  Token Embedding + Positional Embedding (可学习)            │
└─────────────────────────────────────────────────────────────┘
     ↓
┌─────────────────────────────────────────────────────────────┐
│                  Transformer Decoder × N                     │
│  ┌───────────────────────────────────────────────────────┐  │
│  │  Layer Norm (Pre-Norm)                                │  │
│  │  Masked Multi-Head Self-Attention                     │  │
│  │  Residual Connection                                  │  │
│  │  Layer Norm                                           │  │
│  │  Feed-Forward Network                                 │  │
│  │  Residual Connection                                  │  │
│  └───────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
     ↓
┌─────────────────────────────────────────────────────────────┐
│  Final Layer Norm → Linear → Softmax                       │
└─────────────────────────────────────────────────────────────┘
```

### 1.3 InstructGPT 与 RLHF

InstructGPT 引入了 **RLHF（Reinforcement Learning from Human Feedback）** 技术，使模型输出更符合人类意图：

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        RLHF 三阶段训练                                   │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  阶段一: 有监督微调 (SFT)                                                │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  输入: 人工编写的指令-回复对                                       │   │
│  │  目标: 让模型学会遵循指令                                          │   │
│  │  数据量: 约 10K - 100K 高质量样本                                  │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                    ↓                                    │
│  阶段二: 奖励模型训练 (RM)                                               │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  输入: 同一提示词的多个回复                                        │   │
│  │  标注: 人工对回复进行排序                                          │   │
│  │  目标: 训练奖励模型预测人类偏好                                    │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                    ↓                                    │
│  阶段三: 强化学习优化 (PPO)                                              │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  输入: SFT 模型作为策略网络                                        │   │
│  │  目标: 最大化奖励模型的评分                                        │   │
│  │  约束: KL 散度惩罚，防止偏离原始模型                               │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

#### PPO 损失函数

$$
\mathcal{L}(\theta) = -\mathbb{E}\left[\min\left(r_t(\theta)\hat{A}_t, \text{clip}(r_t(\theta), 1-\epsilon, 1+\epsilon)\hat{A}_t\right)\right] + \beta \cdot \mathbb{E}\left[\text{KL}(\pi_\theta || \pi_{\text{ref}})\right]
$$

其中：
- $r_t(\theta) = \frac{\pi_\theta(a_t|s_t)}{\pi_{\text{old}}(a_t|s_t)}$ 是重要性采样比率
- $\hat{A}_t$ 是优势函数估计
- $\epsilon$ 是裁剪参数
- $\beta$ 是 KL 散度惩罚系数

### 2.2 LLaMA 架构创新

LLaMA 在标准 Transformer 架构上进行了多项改进：

#### 架构对比

| 组件 | GPT-3 | LLaMA |
|------|-------|-------|
| **位置编码** | 可学习 | **RoPE（旋转位置编码）** |
| **层归一化** | Pre-LN | **RMSNorm** |
| **激活函数** | GELU | **SwiGLU** |
| **注意力** | MHA | **GQA（LLaMA 2+）** |
| **词表大小** | 50K | 32K |

#### RoPE（旋转位置编码）

RoPE 通过旋转向量来编码位置信息：

$$
\text{RoPE}(x_m, m) = \begin{pmatrix} x_m^{(1)} \\ x_m^{(2)} \\ \vdots \\ x_m^{(d-1)} \\ x_m^{(d)} \end{pmatrix} \odot \begin{pmatrix} \cos(m\theta_1) \\ \cos(m\theta_1) \\ \vdots \\ \cos(m\theta_{d/2}) \\ \cos(m\theta_{d/2}) \end{pmatrix} + \begin{pmatrix} -x_m^{(2)} \\ x_m^{(1)} \\ \vdots \\ -x_m^{(d)} \\ x_m^{(d-1)} \end{pmatrix} \odot \begin{pmatrix} \sin(m\theta_1) \\ \sin(m\theta_1) \\ \vdots \\ \sin(m\theta_{d/2}) \\ \sin(m\theta_{d/2}) \end{pmatrix}
$$

**RoPE 优势**：
- 相对位置感知：两个 token 的注意力只与相对位置有关
- 长度外推：可通过插值扩展到更长序列
- 计算效率：无需学习参数

#### RMSNorm

LLaMA 使用 RMSNorm 替代 LayerNorm：

$$
\text{RMSNorm}(x) = \frac{x}{\sqrt{\frac{1}{d}\sum_{i=1}^d x_i^2 + \epsilon}} \cdot \gamma
$$

相比 LayerNorm，RMSNorm：
- 去掉了均值计算，更高效
- 在 LLM 中效果相当或更好

#### SwiGLU 激活函数

SwiGLU 是 GLU（Gated Linear Unit）的变体：

$$
\text{SwiGLU}(x) = \text{Swish}(xW_1) \odot (xW_2)
$$

其中 $\text{Swish}(x) = x \cdot \sigma(x)$

**优势**：
- 门控机制增强表达能力
- 在 LLM 实验中优于 GELU

#### GQA（分组查询注意力）

LLaMA 2 引入 GQA 以提升推理效率：

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    注意力机制对比                                        │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  MHA (Multi-Head Attention)                                             │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  Q: h 组  ←  K: h 组  ←  V: h 组                                 │   │
│  │  每个头独立 K/V，推理需缓存 h 组 KV                               │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  MQA (Multi-Query Attention)                                            │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  Q: h 组  ←  K: 1 组  ←  V: 1 组                                 │   │
│  │  所有头共享 K/V，推理缓存量最少，但质量下降                        │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  GQA (Grouped-Query Attention)                                          │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  Q: h 组  ←  K: g 组  ←  V: g 组  (g < h)                        │   │
│  │  平衡效率与质量，LLaMA 2 使用 g=8                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 3.5 DeepSeek

DeepSeek 是深度求索公司开源的大语言模型系列。

#### DeepSeek 架构创新

**DeepSeek-MoE** 采用创新的 MoE（混合专家）架构：

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    DeepSeek-MoE 架构                                     │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  传统 MoE:                                                              │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  Router → 选择 Top-k 专家 → 专家计算 → 加权求和                   │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  DeepSeek-MoE:                                                         │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  Router → 共享专家 + 路由专家                                     │   │
│  │                                                                  │   │
│  │  共享专家: 始终激活，捕获通用知识                                 │   │
│  │  路由专家: 选择性激活，捕获专业知识                               │   │
│  │                                                                  │   │
│  │  优势: 减少路由冗余，专家分工更明确                               │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

#### MLA（多头潜在注意力）

DeepSeek-V2 引入 MLA 以大幅降低 KV Cache：

$$
\text{KV Cache} = \text{标准 MHA}: O(n \cdot h \cdot d) \rightarrow \text{MLA}: O(n \cdot d')
$$

其中 $d' \ll h \cdot d$，显著减少推理内存占用。

| **词表大小** | 50K | 32K/128K | 65K/152K | 152K | 100K |

### 4.2 架构设计考量

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    架构设计权衡                                          │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  RoPE vs ALiBi vs 可学习位置编码                                        │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  RoPE: 长度外推好，相对位置感知，主流选择                         │   │
│  │  ALiBi: 外推能力强，无需训练位置编码                              │   │
│  │  可学习: 灵活但外推差，早期 GPT 使用                              │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  MHA vs GQA vs MLA                                                     │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  MHA: 质量最高，但 KV Cache 大                                   │   │
│  │  GQA: 平衡质量与效率，主流选择                                    │   │
│  │  MLA: 最小 KV Cache，DeepSeek 创新                                │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  GELU vs SwiGLU                                                        │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  GELU: 经典选择，GPT/BERT 使用                                    │   │
│  │  SwiGLU: 门控机制，现代 LLM 主流选择                              │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  LayerNorm vs RMSNorm                                                  │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  LayerNorm: 包含均值和方差归一化                                  │   │
│  │  RMSNorm: 仅用均方根，计算更快，现代 LLM 主流                     │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 5.3 部署优化

# 量化部署
from transformers import BitsAndBytesConfig

quantization_config = BitsAndBytesConfig(
    load_in_4bit=True,
    bnb_4bit_compute_dtype=torch.float16,
    bnb_4bit_quant_type="nf4",
    bnb_4bit_use_double_quant=True,
)

model = AutoModelForCausalLM.from_pretrained(
    "Qwen/Qwen2-7B",
    quantization_config=quantization_config,
    device_map="auto"
)

from vllm import LLM, SamplingParams

llm = LLM(model="Qwen/Qwen2-7B")
outputs = llm.generate(["你好，请介绍一下自己"], SamplingParams(max_tokens=100))
