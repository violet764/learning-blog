## 面试准备建议

### 1. 知识体系构建

```
Java 基础
├── 语法基础
├── 面向对象
├── 集合框架
└── 异常处理

并发编程
├── 线程基础
├── 锁机制
├── 线程池
└── 并发工具

JVM
├── 内存模型
├── 垃圾回收
├── 类加载
└── 调优

框架
├── Spring
├── Spring Boot
├── MyBatis
└── Spring Cloud

数据库
├── MySQL
├── Redis
└── 分布式事务
```

### 2. 学习建议

- **理解原理**：不要死记硬背，理解背后的原理
- **动手实践**：写代码验证，加深理解
- **画图总结**：用流程图、架构图梳理知识
- **追问准备**：面试官会追问，准备好追问的答案

### 3. 面试技巧

- 回答结构化：先概括后详细
- 举一反三：关联知识点
- 实战经验：结合项目经历
- 知之为知之：不会就说不会，但要表示学习意愿

## 常见面试题型

### 理论题
- "是什么"：概念解释
- "为什么"：原因分析
- "怎么做"：实现方式

### 对比题
- A 和 B 的区别
- 不同方案的优缺点

### 场景题
- 实际问题如何解决
- 系统设计思路

### 代码题
- 手写实现
- 算法题
