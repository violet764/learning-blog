# 面向对象进阶特性

本文介绍 C++ 面向对象编程的进阶特性，包括继承、多态、抽象类、深浅拷贝等核心概念。这些特性是实现灵活、可扩展代码架构的关键。

## 继承机制

### 继承的基本概念

继承建立了类型之间的 "is-a" 关系，派生类自动获得基类的成员（根据访问控制）。C++ 支持单继承和多继承。

```cpp
// 基类
class Shape {
protected:  // 保护成员：派生类可访问，外部不可访问
    string color;
    
public:
    Shape(const string &c) : color(c) {}
    
    virtual double area() const { return 0; }
    
    void display() const {
        cout << "形状颜色: " << color << endl;
    }
    
    virtual ~Shape() {}  // 虚析构函数
};

// 派生类
class Circle : public Shape {  // public 继承
private:
    double radius;
    
public:
    Circle(const string &c, double r) : Shape(c), radius(r) {}
    
    double area() const override {
        return 3.14159 * radius * radius;
    }
    
    void display() const override {
        Shape::display();  // 调用基类方法
        cout << "圆形半径: " << radius << endl;
    }
};
```

### 继承方式

C++ 支持三种继承方式，决定了基类成员在派生类中的访问权限：

```
┌──────────────────────────────────────────────────────────────────┐
│                      继承方式对访问权限的影响                       │
├─────────────────┬───────────────┬───────────────┬────────────────┤
│    基类成员     │ public 继承   │ protected 继承│ private 继承   │
├─────────────────┼───────────────┼───────────────┼────────────────┤
│ public 成员     │ public        │ protected     │ private        │
│ protected 成员  │ protected     │ protected     │ private        │
│ private 成员    │ 不可访问      │ 不可访问      │ 不可访问       │
└─────────────────┴───────────────┴───────────────┴────────────────┘
```

```cpp
class Base {
public:
    int publicVar = 10;
protected:
    int protectedVar = 20;
private:
    int privateVar = 30;
};

// public 继承：最常用的方式，保持 is-a 关系
class PublicDerived : public Base {
public:
    void test() {
        publicVar = 100;      // ✅ 可访问
        protectedVar = 200;   // ✅ 可访问
        // privateVar = 300;  // ❌ 不可访问
    }
};

// protected 继承：用于实现细节隐藏
class ProtectedDerived : protected Base {
public:
    void test() {
        publicVar = 100;      // ✅ 可访问（变为 protected）
        protectedVar = 200;   // ✅ 可访问
    }
};

// private 继承：用于"实现于"关系
class PrivateDerived : private Base {
public:
    void test() {
        publicVar = 100;      // ✅ 可访问（变为 private）
        protectedVar = 200;   // ✅ 可访问（变为 private）
    }
};
```

### 虚继承与菱形继承问题

当多个派生类继承同一个基类，而这些派生类又被另一个类同时继承时，会产生菱形继承问题：

```
        Animal
        /     \
    Mammal  WingedAnimal
        \     /
          Bat
```

```cpp
// 虚继承解决菱形继承问题
class Animal {
public:
    int age = 0;
    Animal() { cout << "Animal constructor" << endl; }
};

// 使用 virtual 关键字进行虚继承
class Mammal : virtual public Animal {
public:
    Mammal() { cout << "Mammal constructor" << endl; }
};

class WingedAnimal : virtual public Animal {
public:
    WingedAnimal() { cout << "WingedAnimal constructor" << endl; }
};

// Bat 只有一份 Animal 的实例
class Bat : public Mammal, public WingedAnimal {
public:
    Bat() { cout << "Bat constructor" << endl; }
    
    void setAge(int a) {
        age = a;  // ✅ 没有二义性，因为使用了虚继承
    }
};
```

## 多态机制

### 虚函数与动态绑定

多态允许通过基类接口操作派生类对象，在运行时根据实际对象类型调用正确的函数：

```cpp
class Shape {
public:
    // 虚函数：支持运行时动态绑定
    virtual double area() const = 0;  // 纯虚函数
    virtual void display() const {
        cout << "这是一个形状" << endl;
    }
    
    // 虚析构函数：确保正确调用派生类析构函数
    virtual ~Shape() {}
};

class Circle : public Shape {
private:
    double radius;
    
public:
    Circle(double r) : radius(r) {}
    
    double area() const override {
        return 3.14159 * radius * radius;
    }
    
    void display() const override {
        cout << "圆形半径: " << radius << endl;
    }
};

class Rectangle : public Shape {
private:
    double width, height;
    
public:
    Rectangle(double w, double h) : width(w), height(h) {}
    
    double area() const override {
        return width * height;
    }
    
    void display() const override {
        cout << "矩形: " << width << " × " << height << endl;
    }
};

// 多态调用示例
void printArea(const Shape &shape) {
    cout << "面积: " << shape.area() << endl;  // 动态绑定
}

int main() {
    Circle circle(5.0);
    Rectangle rectangle(4.0, 6.0);
    
    printArea(circle);     // 调用 Circle::area()
    printArea(rectangle);  // 调用 Rectangle::area()
    
    // 通过基类指针数组实现多态
    Shape *shapes[] = {&circle, &rectangle};
    for (Shape *shape : shapes) {
        shape->display();  // 动态绑定到具体类型
    }
    
    return 0;
}
```

### 虚函数表（vtable）

每个有虚函数的类都有一个虚函数表，存储虚函数的地址。对象包含指向 vtable 的指针，实现动态绑定：

```
┌─────────────────────────────────────────────────────────┐
│                      虚函数表机制                        │
├─────────────────────────────────────────────────────────┤
│  Shape 对象:                                            │
│  ┌─────────────┐                                        │
│  │ vtable 指针 │ ───→ Shape 的 vtable                   │
│  └─────────────┘     ┌─────────────────────────┐        │
│                      │ &Shape::area            │        │
│                      │ &Shape::display         │        │
│                      │ &Shape::~Shape          │        │
│                      └─────────────────────────┘        │
├─────────────────────────────────────────────────────────┤
│  Circle 对象:                                           │
│  ┌─────────────┐                                        │
│  │ vtable 指针 │ ───→ Circle 的 vtable                  │
│  │ radius      │     ┌─────────────────────────┐        │
│  └─────────────┘     │ &Circle::area           │        │
│                      │ &Circle::display        │        │
│                      │ &Circle::~Circle        │        │
│                      └─────────────────────────┘        │
└─────────────────────────────────────────────────────────┘
```

### override 关键字

C++11 引入 `override` 关键字，确保正确重写基类虚函数：

```cpp
class Base {
public:
    virtual void func() {}
    virtual void func(int x) {}
};

class Derived : public Base {
public:
    void func() override { /* 正确重写 */ }
    // void func(double x) override { }  // ❌ 编译错误：没有重写的函数
};
```

## 纯虚函数与抽象类

### 抽象类的定义

包含纯虚函数的类称为抽象类，不能实例化，专门用于定义接口规范：

```cpp
// 抽象基类：图形接口
class Shape {
public:
    // 纯虚函数：强制派生类实现
    virtual double area() const = 0;
    virtual double perimeter() const = 0;
    virtual void draw() const = 0;
    
    virtual ~Shape() = default;
    
    // 普通成员函数：提供通用功能
    void printInfo() const {
        cout << "面积: " << area() << ", 周长: " << perimeter() << endl;
    }
};

// 具体派生类：圆形
class Circle : public Shape {
private:
    double radius;
    
public:
    Circle(double r) : radius(r) {
        if (radius <= 0) throw invalid_argument("半径必须大于0");
    }
    
    double area() const override {
        return 3.14159 * radius * radius;
    }
    
    double perimeter() const override {
        return 2 * 3.14159 * radius;
    }
    
    void draw() const override {
        cout << "绘制圆形 (半径: " << radius << ")" << endl;
    }
};

// 具体派生类：矩形
class Rectangle : public Shape {
private:
    double width, height;
    
public:
    Rectangle(double w, double h) : width(w), height(h) {}
    
    double area() const override { return width * height; }
    double perimeter() const override { return 2 * (width + height); }
    void draw() const override {
        cout << "绘制矩形 (" << width << " × " << height << ")" << endl;
    }
};
```

### 接口继承示例

```cpp
// 可绘制接口
class Drawable {
public:
    virtual void draw() const = 0;
    virtual ~Drawable() = default;
};

// 可序列化接口
class Serializable {
public:
    virtual string serialize() const = 0;
    virtual void deserialize(const string& data) = 0;
    virtual ~Serializable() = default;
};

// 多重接口继承
class GraphicShape : public Shape, public Drawable, public Serializable {
    // 必须实现所有纯虚函数
};
```

### 抽象类的关键特性

| 特性 | 说明 |
|------|------|
| **纯虚函数** | `virtual 返回类型 函数名() = 0;` |
| **不能实例化** | 抽象类不能创建对象 |
| **虚析构函数** | 多态基类必须有虚析构函数 |
| **接口分离** | 实现接口与实现的解耦 |

## 深拷贝与浅拷贝

### 拷贝语义问题

C++ 默认提供浅拷贝，对于包含动态资源的类会导致问题：

```cpp
// ❌ 浅拷贝问题示例
class ShallowString {
private:
    char* data;
    int length;
    
public:
    ShallowString(const char* str = "") {
        length = strlen(str);
        data = new char[length + 1];
        strcpy(data, str);
    }
    
    // 默认拷贝构造函数（浅拷贝）- 危险！
    // ShallowString(const ShallowString& other) 
    //     : data(other.data), length(other.length) {}
    // 问题：两个对象共享同一块内存
    
    ~ShallowString() {
        delete[] data;  // 双重释放！
    }
};
```

### 深拷贝实现

```cpp
class DeepString {
private:
    char* data;
    int length;
    
public:
    // 构造函数
    DeepString(const char* str = "") {
        length = strlen(str);
        data = new char[length + 1];
        strcpy(data, str);
    }
    
    // 深拷贝构造函数
    DeepString(const DeepString& other) {
        length = other.length;
        data = new char[length + 1];  // 分配新内存
        strcpy(data, other.data);      // 复制内容
    }
    
    // 深拷贝赋值运算符
    DeepString& operator=(const DeepString& other) {
        if (this != &other) {  // 防止自赋值
            char* newData = new char[other.length + 1];
            strcpy(newData, other.data);
            
            delete[] data;      // 释放旧内存
            data = newData;     // 指向新内存
            length = other.length;
        }
        return *this;
    }
    
    // 移动构造函数（C++11）
    DeepString(DeepString&& other) noexcept 
        : data(other.data), length(other.length) {
        other.data = nullptr;  // 置空源对象
        other.length = 0;
    }
    
    // 移动赋值运算符（C++11）
    DeepString& operator=(DeepString&& other) noexcept {
        if (this != &other) {
            delete[] data;
            data = other.data;
            length = other.length;
            other.data = nullptr;
            other.length = 0;
        }
        return *this;
    }
    
    ~DeepString() {
        delete[] data;
    }
};
```

### Rule of Three/Five 法则

```
┌─────────────────────────────────────────────────────────────┐
│                    Rule of Three/Five 法则                   │
├─────────────────────────────────────────────────────────────┤
│  如果需要自定义以下任一函数，通常需要自定义所有函数：          │
│                                                             │
│  Rule of Three (C++03):                                     │
│  1. 析构函数                                                 │
│  2. 拷贝构造函数                                             │
│  3. 拷贝赋值运算符                                           │
│                                                             │
│  Rule of Five (C++11):                                      │
│  4. 移动构造函数                                             │
│  5. 移动赋值运算符                                           │
└─────────────────────────────────────────────────────────────┘
```

### 现代C++解决方案

使用智能指针自动管理资源，避免手动实现拷贝语义：

```cpp
#include <memory>

class ModernString {
private:
    unique_ptr<char[]> data;
    int length;
    
public:
    ModernString(const char* str = "") {
        length = strlen(str);
        data = make_unique<char[]>(length + 1);
        strcpy(data.get(), str);
    }
    
    // 禁用拷贝（unique_ptr 自动管理）
    ModernString(const ModernString&) = delete;
    ModernString& operator=(const ModernString&) = delete;
    
    // 允许移动
    ModernString(ModernString&& other) noexcept = default;
    ModernString& operator=(ModernString&& other) noexcept = default;
    
    const char* getData() const { return data.get(); }
};
```

## 友元机制

友元机制允许特定函数或类访问类的私有成员：

```cpp
class Circle {
private:
    double x, y, radius;
    
public:
    Circle(double x, double y, double r) 
        : x(x), y(y), radius(r) {}
    
    // 声明友元函数
    friend void printCircleInfo(const Circle& c);
    friend double calculateDistance(const Circle& c1, const Circle& c2);
    
    // 声明友元类
    friend class CircleCalculator;
};

// 友元函数定义：可以访问 Circle 的私有成员
void printCircleInfo(const Circle& c) {
    cout << "圆心: (" << c.x << ", " << c.y << ")" << endl;
    cout << "半径: " << c.radius << endl;
}

double calculateDistance(const Circle& c1, const Circle& c2) {
    double dx = c1.x - c2.x;
    double dy = c1.y - c2.y;
    return sqrt(dx * dx + dy * dy);
}

// 友元类
class CircleCalculator {
public:
    static bool isOverlapping(const Circle& c1, const Circle& c2) {
        double distance = calculateDistance(c1, c2);
        return distance < (c1.radius + c2.radius);
    }
};
```

### 友元使用场景

| 场景 | 说明 |
|------|------|
| **运算符重载** | 支持非成员函数形式的运算符重载 |
| **工具函数** | 为类提供额外的辅助功能 |
| **紧密协作类** | 类之间需要深度交互 |

## 运行时类型识别（RTTI）

### dynamic_cast

`dynamic_cast` 用于安全的向下转型：

```cpp
Shape* shape = new Circle(5.0);

// 安全的向下转型
if (Circle* circle = dynamic_cast<Circle*>(shape)) {
    cout << "是圆形，半径: " << circle->getRadius() << endl;
}
```

### typeid

`typeid` 用于获取对象的实际类型信息：

```cpp
#include <typeinfo>

Shape* shape = new Circle(5.0);
cout << "类型: " << typeid(*shape).name() << endl;
```

## 最佳实践总结

### 设计原则

| 原则 | 说明 |
|------|------|
| **优先组合而非继承** | 组合更灵活，耦合度更低 |
| **面向接口编程** | 使用抽象类定义接口 |
| **虚析构函数** | 多态基类必须有虚析构函数 |
| **const 正确性** | 合理使用 const 成员函数 |

### 常见陷阱

```cpp
// ❌ 忘记虚析构函数
class BadBase {
public:
    ~BadBase() {}  // 非虚析构函数
};

class Derived : public BadBase {
    int* data;
public:
    Derived() : data(new int[100]) {}
    ~Derived() { delete[] data; }  // 可能不会被调用！
};

// ✅ 正确做法
class GoodBase {
public:
    virtual ~GoodBase() = default;
};
```
