# 面向对象编程

面向对象编程（Object-Oriented Programming, OOP）是一种以对象为核心的编程范式。C++ 作为一门支持多范式的语言，提供了完整的面向对象特性支持。

## 基本概念

### 核心定义

| 概念 | 说明 |
|------|------|
| **类（Class）** | 创建对象的蓝图或模板，定义了对象的属性和行为 |
| **对象（Object）** | 类的实例，具有状态和行为的实体 |
| **封装（Encapsulation）** | 将数据和操作数据的方法捆绑在一起，隐藏实现细节 |
| **继承（Inheritance）** | 基于已有类创建新类，实现代码复用 |
| **多态（Polymorphism）** | 同一接口，不同实现，运行时动态绑定 |

### 面向对象的四大特性

```
┌─────────────────────────────────────────────────────────────┐
│                      面向对象四大特性                         │
├─────────────┬─────────────┬─────────────┬─────────────────────┤
│    抽象     │    封装     │    继承     │       多态          │
├─────────────┼─────────────┼─────────────┼─────────────────────┤

└─────────────┴─────────────┴─────────────┴─────────────────────┘
```

## 类与对象

### 类的基本定义

```cpp
class 类名 {
访问修饰符:
    // 数据成员（属性）
    数据类型 成员变量;
    
    // 成员函数（方法）
    返回类型 函数名(参数列表);
};  // ⚠️ 注意：类定义必须以分号结尾
```

### 访问修饰符

```cpp
class AccessDemo {
private:    // 🔒 私有成员：仅类内部可访问
    int privateVar;
    
protected:  // 🛡️ 保护成员：类内部和派生类可访问
    int protectedVar;
    
public:     // 🔓 公有成员：任何地方都可访问
    int publicVar;
};
```

| 访问修饰符 | 类内部 | 派生类 | 外部代码 |
|-----------|--------|--------|----------|
| `private` | ✅ | ❌ | ❌ |
| `protected` | ✅ | ✅ | ❌ |
| `public` | ✅ | ✅ | ✅ |

### 基础示例

```cpp
#include <iostream>
#include <string>
using namespace std;

class Person {
public:
    string name;
    int age;
    
    void introduce() {
        cout << "我叫" << name << ", 今年" << age << "岁。" << endl;
    }
    
    void setAge(int newAge) {
        if (newAge > 0 && newAge < 150) {
            age = newAge;
        }
    }
};

int main() {
    Person person1;
    person1.name = "张三";
    person1.setAge(25);
    person1.introduce();
    return 0;
}
```

## 封装性

### 封装的意义

封装将数据和对数据的操作捆绑在一起，通过访问控制实现信息隐藏。

**封装的优点：**
- 数据保护：防止外部代码意外修改内部状态
- 接口清晰：只暴露必要的公有接口
- 实现隐藏：内部实现可以自由修改而不影响使用者

### 封装示例：银行账户类

```cpp
class BankAccount {
private:
    string accountNumber;
    string ownerName;
    double balance;
    
public:
    BankAccount(string accNum, string name, double initialBalance) {
        accountNumber = accNum;
        ownerName = name;
        balance = (initialBalance >= 0) ? initialBalance : 0;
    }
    
    bool deposit(double amount) {
        if (amount > 0) {
            balance += amount;
            return true;
        }
        return false;
    }
    
    bool withdraw(double amount) {
        if (amount > 0 && amount <= balance) {
            balance -= amount;
            return true;
        }
        return false;
    }
    
    double getBalance() const { return balance; }
};
```

## 构造函数与析构函数

### 构造函数类型

```cpp
class Person {
private:
    string name;
    int age;
    
public:
    // 默认构造函数
    Person() : name("Unknown"), age(0) {}
    
    // 参数化构造函数
    Person(const string &n, int a) : name(n), age(a) {}
    
    // 委托构造函数（C++11）
    Person(const string &n) : Person(n, 0) {}
    
    // 拷贝构造函数
    Person(const Person &other) : name(other.name), age(other.age) {}
    
    // 移动构造函数（C++11）
    Person(Person &&other) noexcept 
        : name(std::move(other.name)), age(other.age) {
        other.age = 0;
    }
    
    ~Person() {}
};
```

### 成员初始化列表

```cpp
class Student {
private:
    const int id;       // const成员必须使用初始化列表
    string name;
    int& ageRef;        // 引用成员必须使用初始化列表
    
public:
    Student(int studentId, const string& studentName, int& age) 
        : id(studentId), name(studentName), ageRef(age) {}
};
```

**必须使用初始化列表的情况：**
- const 成员变量
- 引用成员变量
- 没有默认构造函数的类类型成员

### 析构函数

析构函数在对象销毁时自动调用，负责释放资源：

```cpp
class Resource {
private:
    int* data;
    
public:
    Resource() : data(new int[100]) {}
    
    ~Resource() {
        delete[] data;  // 释放动态分配的内存
    }
};
```

## this 指针与静态成员

### this 指针

每个非静态成员函数都隐含一个 `this` 指针参数：

```cpp
class Point {
private:
    int x, y;
    
public:
    Point(int x, int y) {
        this->x = x;  // 解决命名冲突
        this->y = y;
    }
    
    // 链式调用
    Point& setX(int x) {
        this->x = x;
        return *this;
    }
    
    Point& setY(int y) {
        this->y = y;
        return *this;
    }
};

// 使用
Point p(10, 20);
p.setX(30).setY(40);  // 链式调用
```

### 静态成员

静态成员属于类本身而非对象实例：

```cpp
class Counter {
private:
    static int count;  // 所有对象共享
    int id;
    
public:
    Counter() : id(++count) {}
    
    static int getCount() {  // 静态成员函数
        return count;
    }
    
    ~Counter() { --count; }
};

int Counter::count = 0;  // 静态成员定义
```

### const 成员函数

```cpp
class Student {
private:
    string name;
    mutable int accessCount;  // mutable 可在 const 函数中修改
    
public:
    const string& getName() const {
        accessCount++;
        return name;
    }
};
```

## 继承机制

### 继承的基本概念

```cpp
class Shape {
protected:
    string color;
    
public:
    Shape(const string &c) : color(c) {}
    virtual double area() const { return 0; }
    virtual ~Shape() {}
};

class Circle : public Shape {
private:
    double radius;
    
public:
    Circle(const string &c, double r) : Shape(c), radius(r) {}
    
    double area() const override {
        return 3.14159 * radius * radius;
    }
};
```

### 继承方式

| 基类成员 | public 继承 | protected 继承 | private 继承 |
|---------|-------------|----------------|--------------|
| public 成员 | public | protected | private |
| protected 成员 | protected | protected | private |
| private 成员 | 不可访问 | 不可访问 | 不可访问 |

### 虚继承与菱形继承

```cpp
class Animal {
public:
    int age = 0;
};

class Mammal : virtual public Animal {};
class WingedAnimal : virtual public Animal {};

class Bat : public Mammal, public WingedAnimal {
public:
    void setAge(int a) {
        age = a;  // 无二义性
    }
};
```

## 多态机制

### 虚函数与动态绑定

```cpp
class Shape {
public:
    virtual double area() const = 0;  // 纯虚函数
    virtual void display() const {}
    virtual ~Shape() {}
};

class Circle : public Shape {
private:
    double radius;
    
public:
    Circle(double r) : radius(r) {}
    
    double area() const override {
        return 3.14159 * radius * radius;
    }
};

// 多态调用
void printArea(const Shape &shape) {
    cout << "面积: " << shape.area() << endl;  // 动态绑定
}
```

### override 关键字

```cpp
class Base {
public:
    virtual void func() {}
};

class Derived : public Base {
public:
    void func() override { }  // 确保正确重写
};
```

## 深拷贝与浅拷贝

### 拷贝语义问题

```cpp
class DeepString {
private:
    char* data;
    int length;
    
public:
    DeepString(const char* str = "") {
        length = strlen(str);
        data = new char[length + 1];
        strcpy(data, str);
    }
    
    // 深拷贝构造函数
    DeepString(const DeepString& other) {
        length = other.length;
        data = new char[length + 1];
        strcpy(data, other.data);
    }
    
    // 深拷贝赋值运算符
    DeepString& operator=(const DeepString& other) {
        if (this != &other) {
            delete[] data;
            data = new char[other.length + 1];
            strcpy(data, other.data);
            length = other.length;
        }
        return *this;
    }
    
    // 移动构造函数（C++11）
    DeepString(DeepString&& other) noexcept 
        : data(other.data), length(other.length) {
        other.data = nullptr;
        other.length = 0;
    }
    
    ~DeepString() { delete[] data; }
};
```

### Rule of Five 法则

如果需要自定义以下任一函数，通常需要自定义所有函数：
1. 析构函数
2. 拷贝构造函数
3. 拷贝赋值运算符
4. 移动构造函数
5. 移动赋值运算符

## 常见错误

### 1. 忘记类定义的分号

```cpp
class MyClass {
};  // 必须有分号
```

### 2. 忘记虚析构函数

```cpp
class Base {
public:
    virtual ~Base() = default;  // 多态基类必须有虚析构函数
};
```

### 3. 直接访问私有成员

```cpp
class MyClass {
private:
    int secret;
};

MyClass obj;
// obj.secret = 10;  // 错误
```

## 最佳实践

| 原则 | 说明 |
|------|------|
| **优先组合而非继承** | 组合更灵活，耦合度更低 |
| **面向接口编程** | 使用抽象类定义接口 |
| **虚析构函数** | 多态基类必须有虚析构函数 |
| **const 正确性** | 合理使用 const 成员函数 |
| **遵循 Rule of Five** | 管理资源时实现完整的拷贝/移动语义 |
