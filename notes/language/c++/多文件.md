# C++ 项目构建与工程管理

本章将系统介绍C++项目的构建管理，包括多文件编程、编译链接过程、Makefile、CMake以及代码规范等内容。

## 1. 多文件编程基础

### 为什么需要多文件？

**模块化优势**：
- **代码组织**：将相关功能分解为独立模块
- **可重用性**：模块可以在不同项目中复用
- **协作开发**：多人可同时开发不同模块
- **编译效率**：只需重新编译修改过的文件

**典型项目结构**：
```
calculator/
├── include/          # 头文件目录
│   ├── math_ops.h
│   └── utils.h
├── src/              # 源文件目录
│   ├── main.cpp
│   ├── math_ops.cpp
│   └── utils.cpp
├── build/            # 构建输出目录
├── lib/              # 第三方库
├── tests/            # 测试文件
├── Makefile          # 构建脚本
└── CMakeLists.txt    # CMake配置
```

### 头文件与源文件的角色

#### 头文件 (.h/.hpp)

```cpp
// math_operations.h
#ifndef MATH_OPERATIONS_H    // 包含保护
#define MATH_OPERATIONS_H

#include <string>
#include <vector>

// 函数声明
int add(int a, int b);
double add(double a, double b);

// 类定义
class Calculator {
private:
    std::string name;
    
public:
    Calculator(const std::string& calcName);
    
    double multiply(double a, double b) const;
    double divide(double a, double b) const;
    
    // 内联函数可以放在头文件中
    const std::string& getName() const { return name; }
};

// 常量定义
constexpr double PI = 3.141592653589793;
constexpr int MAX_OPERATIONS = 1000;

// 类型别名
using OperationFunc = double(*)(double, double);

// 全局变量声明（使用extern）
extern int globalCounter;

#endif // MATH_OPERATIONS_H
```

**头文件内容规范**：

| 可以放 | 不应该放 |
|--------|----------|
| 函数声明 | 普通函数定义 |
| 类定义 | 全局变量定义 |
| 模板定义 | 大型实现代码 |
| 内联函数 | using namespace |
| 常量定义 | - |
| 类型定义 | - |

#### 源文件 (.cpp)

```cpp
// math_operations.cpp
#include "math_operations.h"  // 包含自己的头文件
#include <stdexcept>
#include <cmath>

// 函数定义
int add(int a, int b) {
    return a + b;
}

double add(double a, double b) {
    return a + b;
}

// 类成员函数定义
Calculator::Calculator(const std::string& calcName) : name(calcName) {}

double Calculator::multiply(double a, double b) const {
    return a * b;
}

double Calculator::divide(double a, double b) const {
    if (b == 0) {
        throw std::invalid_argument("除数不能为零");
    }
    return a / b;
}
```

```cpp
// globals.cpp - 全局变量定义
#include "math_operations.h"

int globalCounter = 0;  // 定义（只能在一个源文件中）
```

## 2. 编译与链接过程

### 四个阶段

```
源文件 (.cpp) → 预处理 → 编译 → 汇编 → 链接 → 可执行文件
```

#### 1. 预处理阶段

```bash
# 查看预处理结果
g++ -E main.cpp -o main.i

# 预处理主要工作：
# - 处理 #include（展开头文件）
# - 处理 #define（宏替换）
# - 处理条件编译（#ifdef, #ifndef）
# - 删除注释
```

#### 2. 编译阶段

```bash
# 生成汇编代码
g++ -S main.cpp -o main.s

# 编译主要工作：
# - 词法分析
# - 语法分析
# - 语义分析
# - 中间代码生成
# - 目标代码生成
```

#### 3. 汇编阶段

```bash
# 生成目标文件
g++ -c main.cpp -o main.o

# 汇编主要工作：
# - 将汇编代码转换为机器码
# - 生成可重定位目标文件
```

#### 4. 链接阶段

```bash
# 链接多个目标文件
g++ main.o math_ops.o utils.o -o calculator

# 链接主要工作：
# - 符号解析
# - 重定位
# - 合并代码段和数据段
```

### 手动编译步骤

```bash
# 步骤1：编译各源文件为目标文件
g++ -c main.cpp -o main.o
g++ -c math_operations.cpp -o math_operations.o
g++ -c utils.cpp -o utils.o

# 步骤2：链接目标文件
g++ main.o math_operations.o utils.o -o calculator

# 步骤3：运行程序
./calculator
```

### 编译选项速查

| 选项 | 说明 | 示例 |
|------|------|------|
| `-c` | 只编译不链接 | `g++ -c file.cpp` |
| `-o` | 指定输出文件名 | `g++ -o prog file.cpp` |
| `-I` | 添加头文件搜索路径 | `g++ -I./include` |
| `-L` | 添加库文件搜索路径 | `g++ -L./lib` |
| `-l` | 链接指定的库 | `g++ -lm`（数学库） |
| `-std` | 指定C++标准 | `g++ -std=c++17` |
| `-Wall` | 开启所有警告 | `g++ -Wall` |
| `-Wextra` | 额外警告 | `g++ -Wextra` |
| `-g` | 生成调试信息 | `g++ -g` |
| `-O0/1/2/3` | 优化级别 | `g++ -O2` |
| `-D` | 定义宏 | `g++ -DDEBUG` |

## 3. Makefile构建系统

### 基本语法

```makefile
# 注释以 # 开头

# 变量定义
CXX = g++
CXXFLAGS = -Wall -g -std=c++17
TARGET = calculator
SOURCES = main.cpp math_operations.cpp utils.cpp
OBJECTS = $(SOURCES:.cpp=.o)

# 默认目标
all: $(TARGET)

# 链接规则
$(TARGET): $(OBJECTS)
	$(CXX) $(CXXFLAGS) -o $@ $^

# 编译规则
%.o: %.cpp
	$(CXX) $(CXXFLAGS) -c $< -o $@

# 清理规则
clean:
	rm -f $(OBJECTS) $(TARGET)

# 伪目标声明
.PHONY: all clean
```

### 自动变量

| 变量 | 含义 | 示例 |
|------|------|------|
| `$@` | 目标文件名 | `$(TARGET)` |
| `$<` | 第一个依赖文件 | `main.cpp` |
| `$^` | 所有依赖文件 | `main.o math_ops.o` |
| `$?` | 比目标新的依赖文件 | 更新的文件 |
| `$*` | 不包含扩展名的目标 | `main` |

### 完整Makefile示例

```makefile
# 编译器设置
CXX = g++
CXXFLAGS = -Wall -Wextra -pedantic -std=c++17 -g

# 项目设置
TARGET = calculator
BUILD_DIR = build
SRC_DIR = src
INCLUDE_DIR = include

# 源文件列表
SOURCES = $(wildcard $(SRC_DIR)/*.cpp)
OBJECTS = $(patsubst $(SRC_DIR)/%.cpp, $(BUILD_DIR)/%.o, $(SOURCES))
DEPENDS = $(OBJECTS:.o=.d)

# 包含路径
CXXFLAGS += -I$(INCLUDE_DIR)

# 默认目标
all: $(BUILD_DIR) $(TARGET)

# 创建构建目录
$(BUILD_DIR):
	@mkdir -p $(BUILD_DIR)

# 链接可执行文件
$(TARGET): $(OBJECTS)
	$(CXX) $(CXXFLAGS) -o $@ $^
	@echo "构建完成: $(TARGET)"

# 编译规则
$(BUILD_DIR)/%.o: $(SRC_DIR)/%.cpp
	$(CXX) $(CXXFLAGS) -c $< -o $@

# 依赖关系（自动生成）
-include $(DEPENDS)

$(BUILD_DIR)/%.d: $(SRC_DIR)/%.cpp
	@$(CXX) -MM $(CXXFLAGS) $< | sed 's|\(.*\)\.o|$(BUILD_DIR)/\1.o $@|' > $@

# 测试目标
test: $(TARGET)
	@echo "运行测试..."
	@./$(TARGET)

# 清理目标
clean:
	rm -rf $(BUILD_DIR) $(TARGET)
	@echo "清理完成"

# 安装目标
install: $(TARGET)
	@cp $(TARGET) /usr/local/bin/
	@echo "安装完成"

# 帮助信息
help:
	@echo "可用目标:"
	@echo "  all      - 构建项目（默认）"
	@echo "  clean    - 清理构建文件"
	@echo "  test     - 运行测试"
	@echo "  install  - 安装到系统"
	@echo "  help     - 显示帮助信息"

.PHONY: all clean test install help
```

### Makefile常用函数

```makefile
# wildcard：匹配文件
SOURCES = $(wildcard src/*.cpp)

# patsubst：模式替换
OBJECTS = $(patsubst src/%.cpp, build/%.o, $(SOURCES))

# addprefix：添加前缀
INCLUDES = $(addprefix -I, include lib/include)

# filter：过滤文件
CPP_FILES = $(filter %.cpp, $(SOURCES))

# shell：执行shell命令
DATE = $(shell date)
```

### 条件判断

```makefile
# 调试/发布模式
DEBUG ?= 0

ifeq ($(DEBUG), 1)
    CXXFLAGS += -DDEBUG -O0 -g
else
    CXXFLAGS += -O2 -DNDEBUG
endif

# 检测操作系统
UNAME_S := $(shell uname -s)
ifeq ($(UNAME_S), Linux)
    LDFLAGS += -lpthread
endif
ifeq ($(UNAME_S), Darwin)
    LDFLAGS += -lpthread
endif
```

## 4. CMake构建系统

### 基础CMakeLists.txt

```cmake
# CMake最低版本要求
cmake_minimum_required(VERSION 3.10)

# 项目名称和基本信息
project(MyProject 
    VERSION 1.0.0
    DESCRIPTION "一个示例C++项目"
    LANGUAGES CXX
)

# 设置C++标准
set(CMAKE_CXX_STANDARD 17)
set(CMAKE_CXX_STANDARD_REQUIRED ON)
set(CMAKE_CXX_EXTENSIONS OFF)

# 编译器选项
if(CMAKE_CXX_COMPILER_ID MATCHES "GNU|Clang")
    add_compile_options(-Wall -Wextra -Wpedantic)
elseif(MSVC)
    add_compile_options(/W4)
endif()

# 添加可执行文件
add_executable(my_app 
    src/main.cpp
    src/math_ops.cpp
    src/utils.cpp
)

# 包含目录
target_include_directories(my_app 
    PRIVATE 
        ${CMAKE_CURRENT_SOURCE_DIR}/include
)

# 链接库
target_link_libraries(my_app 
    PRIVATE 
        pthread
        m
)

# 安装规则
install(TARGETS my_app RUNTIME DESTINATION bin)
```

### 多目录项目

```cmake
# 根目录 CMakeLists.txt
cmake_minimum_required(VERSION 3.10)
project(MyProject)

add_subdirectory(src)
add_subdirectory(lib)
add_subdirectory(tests)

# src/CMakeLists.txt
add_library(mylib STATIC
    math_ops.cpp
    utils.cpp
)

target_include_directories(mylib PUBLIC include)

add_executable(myapp main.cpp)
target_link_libraries(myapp PRIVATE mylib)

# tests/CMakeLists.txt
enable_testing()

add_executable(test_math test_math.cpp)
target_link_libraries(test_math PRIVATE mylib)

add_test(NAME MathTest COMMAND test_math)
```

### 查找和使用外部库

```cmake
# 查找系统库
find_package(Threads REQUIRED)
find_package(OpenSSL REQUIRED)
find_package(Boost REQUIRED COMPONENTS filesystem system)

target_link_libraries(myapp 
    PRIVATE 
        Threads::Threads
        OpenSSL::SSL
        OpenSSL::Crypto
        Boost::filesystem
        Boost::system
)

# 自定义查找库
find_path(MYLIB_INCLUDE_DIR mylib.h 
    PATHS /usr/local/include /opt/include
)

find_library(MYLIB_LIBRARY mylib 
    PATHS /usr/local/lib /opt/lib
)

if(MYLIB_INCLUDE_DIR AND MYLIB_LIBRARY)
    target_include_directories(myapp PRIVATE ${MYLIB_INCLUDE_DIR})
    target_link_libraries(myapp PRIVATE ${MYLIB_LIBRARY})
endif()
```

### 构建类型与选项

```cmake
# 构建类型
if(NOT CMAKE_BUILD_TYPE)
    set(CMAKE_BUILD_TYPE Release CACHE STRING "Build type" FORCE)
endif()

set(CMAKE_CXX_FLAGS_DEBUG "-g -O0 -DDEBUG")
set(CMAKE_CXX_FLAGS_RELEASE "-O2 -DNDEBUG")

# 自定义选项
option(BUILD_TESTS "Build tests" ON)
option(USE_OPENMP "Use OpenMP" OFF)

if(BUILD_TESTS)
    enable_testing()
    add_subdirectory(tests)
endif()

if(USE_OPENMP)
    find_package(OpenMP REQUIRED)
    target_link_libraries(myapp PRIVATE OpenMP::OpenMP_CXX)
endif()
```

## 7. 代码规范

### 命名约定


## 8. 常见错误与解决方案

### 编译错误

```cpp
// 错误1：重复定义
// 原因：函数定义放在头文件中，被多个源文件包含
// 解决：头文件只放声明，定义放在源文件

// 错误2：未定义引用
// 原因：链接时找不到函数定义
// 解决：确保链接所有目标文件和库

// 错误3：头文件找不到
// 原因：包含路径不正确
// 解决：使用 -I 指定头文件路径

// 错误4：类型不完整
// 原因：只有声明没有定义
class ForwardDeclared;  // 前向声明
ForwardDeclared obj;    // 错误：类型不完整
ForwardDeclared* ptr;   // 正确：指针可以
```

### 链接错误

```bash
# 错误：undefined reference to `func'
# 原因：函数声明了但未定义，或未链接对应的库
# 解决：检查函数定义，确保链接所有目标文件

# 错误：multiple definition of `var'
# 原因：全局变量在头文件中定义
// 错误写法（header.h）
int globalVar = 0;  // 不要这样写

// 正确写法
// header.h
extern int globalVar;  // 声明

// source.cpp
int globalVar = 0;  // 定义
```

### 运行时错误

```cpp
// 1. 数组越界
std::vector<int> vec = {1, 2, 3};
int val = vec.at(5);  // 抛出异常
// int val = vec[5];  // 未定义行为

// 2. 空指针解引用
int* ptr = nullptr;
// *ptr = 10;  // 运行时错误
if (ptr) *ptr = 10;  // 安全检查

// 3. 内存泄漏
void leak() {
    int* p = new int(42);
    // 忘记delete
}
// 解决：使用智能指针

// 4. 悬空指针
int* dangling() {
    int local = 42;
    return &local;  // 返回局部变量地址
}
```
